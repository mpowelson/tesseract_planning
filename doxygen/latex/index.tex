The planning server is built leveraging the library taskflow which enables you to quickly write parallel and heterogeneous tasks programs which aligns well with process motion planning. These types of libraries leverage thread pools to quickly spin up different tasks as needed. One interesting thing found, if a task consumes a lot of memory, then the malloc library can make a performance decision to allow the thread to hold onto the memory it has for subsequent tasks opposed to giving the memory back to your system. This may be fine if this was the only set of tasks being run but typically in our use case there are other computationally intensive processes which could use this memory.\hypertarget{index_autotoc_md6}{}\doxysubsection{Ubuntu}\label{index_autotoc_md6}
\hypertarget{index_autotoc_md7}{}\doxysubsection{tcmalloc (\+Recommend)}\label{index_autotoc_md7}
The recommended approach to address memory management is to leverage the {\ttfamily tcmalloc} library which is part of Google Performance Tools ({\ttfamily gperftools}). Experimentation has shown {\ttfamily tcmalloc} to provide more stable memory management when leveraging the planning server.

This capability can be leveraged by linking the exectuable that contains the planning server against {\ttfamily tcmalloc} using the following commands\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{find\_package(tesseract\_common REQUIRED)}
\DoxyCodeLine{find\_package(tcmalloc REQUIRED)}
\DoxyCodeLine{}
\DoxyCodeLine{add\_executable(my\_executable src/my\_executable.cpp)}
\DoxyCodeLine{target\_link\_libraries(my\_executable PUBLIC tcmalloc::tcmalloc)}

\end{DoxyCode}


This library provides a set of run-\/time \href{https://gperftools.github.io/gperftools/tcmalloc.html}{\texttt{ tunables}}, leveraging a set of environment variables.

The one parameter that has been used to configure tcmalloc to give memory back to the OS is {\ttfamily TCMALLOC\+\_\+\+RELEASE\+\_\+\+RATE}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{TCMALLOC\_RELEASE\_RATE   default: 1.0    Rate at which we release unused memory to the system, via madvise(MADV\_DONTNEED), on systems that support it. Zero means we never release memory back to the system. Increase this flag to return memory faster; decrease it to return memory slower. Reasonable rates are in the range [0,10].}

\end{DoxyCode}


Note\+: This is typically set in the roslaunch file for the planning server node leveraging the {\ttfamily $<$env name=\char`\"{}\+TCMALLOC\+\_\+\+RELEASE\+\_\+\+RATE\char`\"{} value=\char`\"{}10\char`\"{}$>$} tag.\hypertarget{index_autotoc_md8}{}\doxysubsection{$<$tt$>$ptmalloc$<$/tt$>$ (\+Ubuntu OS default)}\label{index_autotoc_md8}
The great thing is, the default malloc library {\ttfamily ptmalloc} provides run-\/time \href{https://www.gnu.org/software/libc/manual/html_node/Tunables.html}{\texttt{ tunables}} through a set of environment variables.

The one parameter that has been used to configure libc so it always gives back the memory to the system is setting following memory tunnable\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Tunable: glibc.malloc.mmap\_threshold}
\DoxyCodeLine{}
\DoxyCodeLine{export GLIBC\_TUNABLES=glibc.malloc.mmap\_threshold=1000000}
\DoxyCodeLine{}
\DoxyCodeLine{This tunable supersedes the MALLOC\_MMAP\_THRESHOLD\_ environment variable and is identical in features.}
\DoxyCodeLine{}
\DoxyCodeLine{When this tunable is set, all chunks larger than this value in bytes are allocated outside the normal heap, using the mmap system call. This way it is guaranteed that the memory for these chunks can be returned to the system on free. Note that requests smaller than this threshold might still be allocated via mmap.}
\DoxyCodeLine{}
\DoxyCodeLine{If this tunable is not set, the default value is set to ‘131072’ bytes and the threshold is adjusted dynamically to suit the allocation patterns of the program. If the tunable is set, the dynamic adjustment is disabled and the value is set as static.}

\end{DoxyCode}


Note\+: This is typically set in the roslaunch file for the planning server node leveraging the {\ttfamily $<$env name=\char`\"{}\+GLIBC\+\_\+\+TUNABLES\char`\"{} value=\char`\"{}glibc.\+malloc.\+mmap\+\_\+threshold=1000000\char`\"{}$>$} tag.\hypertarget{index_autotoc_md9}{}\doxysubsection{Windows}\label{index_autotoc_md9}
